多任务学习（MTL）的背景

多任务学习的经典问题是Argyriou et al于08年的提出的预测多个学校的学生成绩。
在研究这个问题中发现，使用多任务学习的预测方法的效果比传统的 Ridge Regression 和 Lasso的效果好很多。
多任务学习不同于单任务学习，它可以同时处理多个任务并捕获内在的关联。

多任务学习 > 多标签学习 > 多类别学习

举个例子，对于网页分类问题：一个网页对应多个类别，不同类别之间具有潜在的相关性。

多任务学习可以用于投资组合，协同合作，网页图片视频检索，疾病建模，疾病预测等

多任务学习的范式一般为：
$$
\min_{w}L \left( w \right) + \Omega \left( w \right)
$$

即最小化损失函数和正则项

MTL 分类（根据不同任务之间是否具有相关性）

MTL的一个基本假设是某些任务都是相互关联的。那么怎么将任务的关联表示出来呢？
方法有以下几种：（以下的5种情况认为所有任务是相互关联的）

  1. Mean-regularized MTL
  2. Joint feature learning
  3. Trace-Norm regularized MTL
  4. Alternating structural optimization（ASO）
  5. Shared Parameter Gaussian Process

有些情况下并不是所有的任务都是相互关联的，还有许多outliers，（事实上这更符合实际的情况），

  1. robust MTL (处理具有离群task的情况)

或者一些情况下不同聚类之间任务是相互关联的，这时候需要使用如下的几种方法来刻画多任务之间的关联性：

  1. Clustered MTL （Tasks have group structures）
  2. Tree MTL (Tasks have treee structures)
  3. Network MTL (Tasks have graph/network structures)

根据上图，我们可以知道，多任务学习的主要目标就是利用原始数据和样本学习一个映射矩阵。

Mean-regularized MTL

这个模型的假设是：所有任务的目标参数向量互相比较接近。
这个假设的优点是简单易实现，缺点是不能完全反映真实情况；

高维特征空间的MTL

embedded feature selection L1/Lq —— group lasso

MTL with Joint Feature Learning

一种获取多任务相关性的方法是限制所有的模型来共享相同的特征集
例如：在school data中，不同学校学生的得分可以使用相似的特征集来确定。


事实表明，使用这种联合特征学习的方法确实可以达到比单任务学习更好的效果。

Dirty Model for MTL

在实际应用中，限制所有的任务共享单个的结构未免太过严格

Robust Multi-Task Feature Learning（RMTL）

这是第二类方法，主要是为了处理那些离群的特殊task。
同时获取相关任务的公共特征集，并检验出异常的task

低阶子空间学习

通过共享的低阶结构来捕获任务之间的相关性

使用正则项来捕获聚类结构
在K均值聚类中，使用最小化均方差的方法来捕获结构。

具有树状结构的多任务学习

在一些应用中，任务之间的关系可以用树状图来表示：属于同一个节点的任务彼此比较相近。相同深度的节点具有相关性。

实际使用中的技巧

MTL vs STL
当训练样本数目比较小时，使用MTL效果更好
Shared features vs shared subspace
当数据维度很大时，识别共享特征要更好
当任务数目很多时，识别共享子空间要更好。

MTL工具箱使用

对于最简单的lasso，可将下式中的 $$$ \rho L2 $$$ 设为 0
$$
\min{w} \sum{i=1}^{t} \left| W{j}^{T}X{j} -Yj \right| + \rho{1} \left| W \right|{1}+\rho L2 \left| W \right|{F}^{2}
$$

输入数据：

X：<1*139 cell>   每个cell： 200*28 double  91*28 double ...
Y: <1*139 cell>   每个cell： 200*1 double   91*1 double ...

关键参数：

初始的一些参数
opts.init = 0;  % guess start point from data. 
opts.tFlag = 1;     % terminate after relative objective value does not changes much.
opts.tol = 10^-5;   % tolerance. 
opts.maxIter = 1500; % maximum iteration number of optimization.


在训练中
[W funcVal] = Least_Lasso(X, Y, lambda(i), opts); % 一般形式的优化函数形式
opts.init = 1; %set the solution as the next initial point. this gives better efficiency. 注意：这个 init=1 必须和 W0 配合使用才能生效。
opts.W0 = W;
